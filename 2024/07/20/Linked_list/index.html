  • 23.Merge k Sorted Lists:
    • similar to 21, define its solution as a new function mergeTwo, then call it in each two of the linked lists.

  • 86.Partition List:
    • Use p1 and p2 to separately connect the nodes smaller than X and larger than X. Pointer p serve as a needle.
    • Note that the solution of this question is different, as we need to disconnect p and p.next, otherwise the result will “contain a ring”. Pay attention to the use of tmp.

2. Slow & Fast Pointers

  • 19.Remove Nth Node From End of List:
    • Let fast pointer move N+1 first, then let the slow one start to move.
    • Note that if you use while fast: to loop, fast will move to None at the end of the linked list, so you need to let fast move N + 1 instead of N.
  • 876.Middle of the Linked List:
    • similar to 19, it focus on finding the node corresponding to a certain idx in the linked list, but this time the idx is not given.
    • It can be solved by moving fast two steps while moving slow one step.
    • Another solution is to calculate the number n of the nodes, and then traverse the linked lists for n/2 times. Remember to treat n differently if n%2!=0.
  • 60.Intersection of Two Linked Lists:
    • First find out the length of the two linked lists, and then let the p of the longer linked list first move to the node that makes the length equal to one another.
